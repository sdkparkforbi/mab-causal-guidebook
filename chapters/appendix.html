GitHub · Vercel · Streamlit · 트러블슈팅

각 장에서 “부록 참고”라고 한 실무 절차 — 저장소 만들기부터 웹앱 공개까지를 한곳에 모았습니다.

배포 정책 — 정적은 Vercel, 검증(DB)은 Streamlit 정적 앱(그림책·시각화)은 Vercel로 통일하고, 검토 앱(HITL)은 DB 저장이 필요해 Streamlit Cloud로 배포합니다.
  • 📖 그림책 — mab-causal-guidebook → Vercel (현재 Pages 배포됨)
  • 🕸️ 시각화 — mab-causal-network → Vercel (현재 Pages 배포됨)
  • ✅ 검증 앱(HITL) — mab-review-app (Streamlit + DB) → Streamlit Cloud
※ 정적 lite 검토는 mab-causal-review(localStorage)도 가능. review-validation(아마존 리뷰)은 무관.

왜 Streamlit은 Vercel에 못 올리나요? Vercel은 “요청이 오면 잠깐 실행되는” 정적·서버리스 호스팅입니다. Streamlit은 항상 켜져 웹소켓으로 상태를 유지하는 앱이라 맞지 않아, 전용 호스팅(Streamlit Cloud)을 씁니다.

B. Git 작업 흐름 — 방향이 두 개

“클론 후 작업”은 GitHub에 repo가 이미 있을 때의 흐름입니다. 반대로 이 그림책처럼 로컬에 먼저 만든 폴더는 “올리기” 방향입니다. 둘 다 익혀두세요.

방향 A — 로컬 폴더를 GitHub에 처음 올리기 (이 그림책)

PowerShell
cd D:\Teaching\2026_1\GenAIBiC\14주차\mab-causal-guidebook
git init
git add .
git commit -m "그림책 초안: 0~3장 + 부록"

# gh CLI가 있으면 repo 생성 + 푸시를 한 번에
gh repo create mab-causal-guidebook --public --source=. --push

# 또는 웹에서 빈 repo를 먼저 만들었다면
git remote add origin https://github.com/<아이디>/mab-causal-guidebook.git
git branch -M main
git push -u origin main

방향 B — GitHub repo를 받아서 작업 (협업·다른 기기)

PowerShell
git clone https://github.com/<아이디>/mab-causal-guidebook.git
cd mab-causal-guidebook
# ...파일 수정...
git add .
git commit -m "2장 작성"
git push
한 번 연결되면 그 뒤로는 루프 수정 → git add → git commit → git push 만 반복하면 됩니다.

C. Vercel 배포 — 그림책 · 시각화 앱(정적)

Vercel은 “GitHub repo를 연결해두면 push할 때마다 자동으로 다시 배포”하는 정적 호스팅입니다. 순수 HTML이라 빌드 설정이 거의 없습니다.

방법 1 — 대시보드 Import (가장 쉬움, 권장)

1

로그인

vercel.com에 GitHub 계정으로 로그인.

2

Import

Add New ▸ Project ▸ Import에서 해당 repo 선택.

3

설정

Framework Preset = Other(정적). Root Directory = index.html이 있는 폴더.

4

Deploy

https://<프로젝트>.vercel.app 발급. 이후 git push만 하면 자동 재배포.

방법 2 — CLI

PowerShell
npm i -g vercel
cd mab-causal-guidebook
vercel          # 처음: 로그인 + 프로젝트 연결 → 미리보기 URL
vercel --prod   # 운영(production) URL로 배포

대안 — GitHub Pages

결과물은 비슷합니다. Settings ▸ Pages ▸ Branch main / root 선택 → 저장 → https://<아이디>.github.io/<저장소>/. index.html이 루트에 있어야 합니다.

Vercel vs Pages Vercel은 push마다 미리보기 URL을 따로 주고 커스텀 도메인·속도가 강점, Pages는 GitHub 안에서 끝나 더 단순. 정적 사이트면 둘 다 무료 — 수업에서는 시각화 앱은 Vercel, 그림책은 Pages로 나눠 둘 다 경험해 보길 권합니다.

D. Streamlit Community Cloud — 검증 앱(5장)

검증 앱은 app.py로 도는 파이썬 서버라 전용 호스팅을 씁니다.

저장소 구조 (repo ③)

mab-review-app/
mab-review-app/
├── app.py              # Streamlit 앱 본체
├── requirements.txt    # streamlit, openai, pandas ... (의존성)
└── .streamlit/
    └── secrets.toml    # ⚠ 로컬 전용 — .gitignore로 제외! (키는 클라우드 UI에 입력)

배포 단계

1

push

위 구조를 GitHub repo에 올립니다.

2

New app

share.streamlit.io → GitHub 로그인 → New app.

3

지정

repo / branch / app.py 경로 선택 → Deploy.

4

공개

https://<앱이름>.streamlit.app 발급. push마다 자동 재배포.

비밀값(키·DB 비밀번호) 처리 — 중요

코드나 repo에 절대 넣지 않습니다. 대시보드 App settings ▸ Secrets에 붙여넣고 코드에서 읽습니다.

app.py
import streamlit as st
api_key = st.secrets["OPENAI_API_KEY"]   # 클라우드 Secrets에서 안전하게
대안 호스팅 Hugging Face Spaces, Render에도 올릴 수 있지만, 학습용으론 Streamlit Cloud가 가장 간단합니다.

E. 어떤 걸 고를까 — 한 장 요약

cheatsheet
정적(HTML)   → Vercel(통일) 또는 Pages   ← 그림책 · 시각화 · (검증 lite)

F. DB 없이 진행하는 무료 대안

  • MySQL 대신 SQLite(파일 한 개) 또는 CSV로 저장 — 이 그림책의 2·3장은 이미 CSV 경로로 제공.
  • 각 장의 입출력을 파일명으로 잇기: step1_*.csv → step2_*.csv → step3_edges.csv.
  • SQLite 예: import sqlite3; con = sqlite3.connect("mab.db") 후 df.to_sql(...).

G. OpenAI API 키 발급·관리

  • platform.openai.com에서 키 발급 → 사용량 한도 설정으로 과금 사고 방지.
  • Colab은 Secret(🔑), 로컬은 환경변수, Streamlit은 Secrets에 보관.
  • 비용 절감: 저비용 모델(gpt-4o-mini), 배치 호출, 결과 캐싱(이미 처리한 PMID 건너뛰기).
