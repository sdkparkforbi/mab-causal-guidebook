CHAPTER 3

인과관계 추출 · 정규화 · 엣지 집계

초록 한 편 한 편에서 원인 → 결과를 캐내, 우리 지식그래프의 ‘뼈대’를 만듭니다.

이 장의 결과물 step3_edges.csv — 고유 엣지 약 3.3만 개 (원인·결과·관계유형·빈도·논문수·근거). 4장 시각화와 6장 모델의 입력입니다.
실제 연구 규모 (정본) 정본 시스템은 5,576편에서 22-relation 스키마로 SVO 트리플을 추출해 32,939개 directed edge · 8,670개 노드(출처 논문 provenance 보존)를 만듭니다. 허브 노드는 aggregation · binding activity · immunogenicity — SC 전환 실패 모드와 일치합니다.

1. 큰 그림 — 세 번에 나눠 짓는다

한 번에 깔끔한 그래프가 나오지 않습니다. 세 단계로 나눕니다.

2. 공통 어휘 — 6개 카테고리, 22개 관계

추출이 제멋대로면 그래프가 누더기가 됩니다. 그래서 모든 노드는 6색 카테고리 중 하나로, 모든 화살표는 22개 관계 중 하나로만 분류하도록 약속합니다.

카테고리뜻예시
sequence아미노산 서열 특징CDR Asn, mutation, 당화 부위
structure3차 구조 특징소수성 패치, Fab 도메인, 이황화결합
formulation제형 조건PS80, sucrose, pH, 농도
stress환경 스트레스열, 교반, 동결-융해
stability안정성 속성·열화응집, 탈아미드화, Tm, 점도
quality_outcome측정 가능한 품질 결과입자 형성, 면역원성, 전하변이

관계는 극성(좋음/나쁨/중립)으로 묶입니다.

긍정: stabilizes · inhibits · prevents · decreases · shields
부정: destabilizes · promotes · increases · induces · aggregates · oxidizes · deamidates · …
중립: correlates · modifies · binds · requires

극성 규칙 — 헷갈리는 포인트 increases는 나쁜 것이 늘 때만(응집 증가), decreases는 나쁜 것이 줄 때만(점도 감소) 씁니다. 그래야 “녹색=안정에 좋음 / 빨강=나쁨”이라는 색 약속이 그래프 전체에서 일관됩니다.

3. 2a 단계 — GPT로 인과관계 자유 추출

전문가 프롬프트로 초록에서 관계를 뽑습니다. 두 가지 규칙이 품질을 크게 좌우합니다.

  • 매개 사슬: A가 C를 거쳐 B에 영향을 주면 A→C와 C→B를 둘 다 뽑습니다.
  • 조건 구체화: 그냥 “pH”가 아니라 low pH, “temperature”가 아니라 elevated temperature.
cell · extraction prompt (발췌)
EXTRACTION_SYSTEM_PROMPT = """You are an expert pharmaceutical scientist
specializing in monoclonal antibody (mAb) formulation and stability.

Extract MECHANISTIC relationships relevant to mAb STABILITY from the abstract.

=== 6 NODE CATEGORIES ===
sequence / structure / formulation / stress / stability / quality_outcome

=== 22 RELATION TYPES ===
Positive : stabilizes, inhibits, prevents, decreases, shields
Negative : destabilizes, promotes, increases, induces, aggregates,
           oxidizes, deamidates, isomerizes, fragments, unfolds,
           adsorbs, precipitates, degrades
Neutral  : correlates, modifies, binds, requires

=== POLARITY RULES ===
"increases" = only bad things increasing (e.g., aggregation increases)
"promotes"  = only bad processes (e.g., promotes aggregation)

=== MEDIATION CHAIN ===
When A affects B through mediator C, extract BOTH:  A -> C  AND  C -> B

=== CONDITION SPECIFICITY ===
Use specific terms: "low pH" not "pH", "elevated temperature" not "temperature"

=== OUTPUT (JSON only) ===
{"relations": [
  {"cause":"...", "category_cause":"...",
   "effect":"...", "category_effect":"...",
   "relationship":"...", "confidence":"high/medium/low",
   "evidence":"key phrase from abstract (max 30 words)"}
]}"""
cell · extract_relations
def extract_relations(title, abstract):
    """초록 한 편 → 관계 리스트"""
    user = f"Title: {title}\nAbstract: {abstract}"
    for attempt in range(3):
        try:
            resp = client.chat.completions.create(
                model="gpt-5-mini",
                messages=[{"role":"system","content":EXTRACTION_SYSTEM_PROMPT},
                          {"role":"user","content":user}],
                max_completion_tokens=10000)
            txt = resp.choices[0].message.content.strip()
            txt = txt.replace("```json","").replace("```","").strip()
            return json.loads(txt).get("relations", [])
        except Exception:
            time.sleep(2 + attempt*2)
    return []

# relevant=1 논문 전체를 돌며 원시 관계를 모은다 (raw)
df = pd.read_csv(f"{PROJECT_DIR}/step2_filtered.csv")
df = df[df.relevant == 1]
raw = []
for _, r in tqdm(df.iterrows(), total=len(df), desc="Extracting"):
    for rel in extract_relations(r.title, r.abstract):
        rel["pmid"] = r.pmid; raw.append(rel)
    time.sleep(0.3)
df_raw = pd.DataFrame(raw)
print(f"원시 관계: {len(df_raw):,}개")
예시 출력원시 관계: 39,792개 (relevant 논문 5,576편 기준)
신뢰성을 위한 작은 습관 각 관계에 evidence(초록 속 근거 문장)와 confidence를 함께 저장합니다. 나중에 5장 검증 앱에서 “이 관계의 근거가 뭐였지?”를 바로 확인할 수 있습니다.

4. 2b 단계 — 용어 정규화(표현 통일)

같은 개념을 사람마다 다르게 씁니다. “high protein concentration”, “elevated mAb concentration”, “concentrated antibody solution” → 전부 high concentration 하나로 합쳐야 그래프에서 한 점으로 모입니다. 표준 용어 159개를 기준표로 주고 GPT가 매핑하게 합니다.

cell · normalization (개념)
NORMALIZATION_SYSTEM_PROMPT = f"""You are an expert in mAb stability terminology.
Map each input term to the most appropriate canonical term from the list.
If none fits, keep it concise and lowercase.

CANONICAL TERMS: {', '.join(canonical_list)}   # 159개 표준 용어

RULES:
- synonyms/variants → canonical  (e.g., "protein aggregation" -> "aggregation")
- keep condition-specific terms distinct  (e.g., "low pH" vs "high pH")
Return ONLY JSON: {{"original term": "canonical term", ...}}"""

# cause/effect 표현을 모아 배치로 표준어에 매핑
cause_map  = normalize_terms_batch(df_raw["cause"].tolist())
effect_map = normalize_terms_batch(df_raw["effect"].tolist())
df_raw["cause_std"]  = df_raw["cause"].map(lambda x: cause_map.get(str(x), x))
df_raw["effect_std"] = df_raw["effect"].map(lambda x: effect_map.get(str(x), x))

5. 2c 단계 — 엣지로 집계

이제 같은 (원인_표준 → 결과_표준) 쌍을 모두 모아 하나의 엣지로 만듭니다. 이때 빈도(몇 번 등장했나)와 논문수(몇 편이 말했나), 대표 관계유형·대표 근거를 함께 계산합니다. “여러 논문이 반복해서 말한 관계일수록 굵은 화살표”가 되는 거죠.

cell · aggregate edges
from collections import Counter, defaultdict

agg = defaultdict(lambda: {"freq":0, "pmids":set(), "rels":Counter(), "ev":[]})
for _, r in df_raw.iterrows():
    key = (r.cause_std, r.effect_std)
    agg[key]["freq"]  += 1
    agg[key]["pmids"].add(str(r.pmid))
    agg[key]["rels"][r.relationship] += 1
    if str(r.get("evidence","")): agg[key]["ev"].append(str(r.evidence)[:120])

edges = [{"cause":c, "effect":e,
          "relationship": d["rels"].most_common(1)[0][0],
          "frequency": d["freq"], "num_papers": len(d["pmids"]),
          "sample_evidence": d["ev"][0] if d["ev"] else ""}
         for (c,e), d in agg.items()]

df_edges = pd.DataFrame(edges).sort_values("frequency", ascending=False)
df_edges.to_csv(f"{PROJECT_DIR}/step3_edges.csv", index=False, encoding="utf-8-sig")
print(f"고유 엣지: {len(df_edges):,}개")
예시 출력고유 엣지: 32,939개 --- 관계유형 분포 --- modifies 7,431 · correlates 5,529 · decreases 3,657 stabilizes 2,823 · increases 2,408 · induces 2,306 ... --- 빈도 상위 엣지 --- thermal stress --[promotes]--> aggregation (freq 29, papers 28) aggregation --[induces]--> immunogenicity (freq 52, papers 41) fcrn binding --[stabilizes]-> half-life (freq 33, papers 27) self-association--[increases]--> viscosity (freq 25, papers 23) excipients --[inhibits]--> aggregation (freq 26, papers 26)

6. 결과 읽기 — 그래프가 말하는 것

상위 엣지만 봐도 안정성의 핵심 줄거리가 드러납니다.

  • thermal stress → aggregation: 열이 응집을 부른다(가장 고전적 경로).
  • aggregation → immunogenicity: 응집이 면역원성으로 이어진다 → SC 전환에서 가장 경계할 사슬.
  • excipients ⊣ aggregation: 부형제가 응집을 막는다(처방으로 개입 가능한 지점).
  • self-association → viscosity: 고농도에서 항체끼리 뭉쳐 점도가 오른다.

이 줄거리들이 4장에서 색과 굵기를 가진 그림으로, 6장에서 예측 가능한 패턴으로 살아납니다.

주의 — 정규화·집계 셀은 한 번만 원본 노트북에서 2b·2c는 결과 테이블을 TRUNCATE(비우고 다시 채움)합니다. CSV로 따라 할 때도 같은 입력으로 한 번만 실행하세요. 추출(2a)은 ‘이어하기’가 되지만, 집계는 전체를 다시 만듭니다.

오늘의 마무리

  • 6 카테고리·22 관계라는 공통 어휘로 추출을 길들였습니다.
  • 추출(2a) → 정규화(2b) → 집계(2c)로 ~3.3만 개 엣지의 그래프 뼈대를 완성했습니다.
  • 빈도 상위 엣지로 안정성의 핵심 줄거리를 읽었습니다.
