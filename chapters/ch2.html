CHAPTER 2

GPT로 관련성 필터링하기

1장에서 모은 ~2,100편 중 “진짜로 mAb 안정성 메커니즘을 다루는” 논문만 GPT에게 골라내게 합니다.

이 장의 결과물 step2_filtered.csv — 각 논문에 relevant(1/0)과 GPT가 남긴 한 줄 판정 이유가 붙은 표. 3장에서는 relevant=1만 사용합니다.
실제 연구 규모 — 그리고 모델에 따라 엄격함이 다르다 실제 논문은 62,281편을 필터링해 7,548편(12.1%)을 남겼습니다(GPT-4o-mini, 약 21시간). 더 엄격한 GPT-5-mini로 다시 돌리면 5,576편(9.0%)만 남아, 도구로만 쓰인 논문·임상 효능 논문을 더 충실히 걸러냅니다.

1. 왜 필터가 필요한가 — 체로 거르기

“monoclonal antibody” 가 들어갔다고 다 안정성 논문은 아닙니다. 항체를 단지 검출 시약(ELISA)으로 쓴 논문, 안정성 얘기는 없는 임상 효능 논문도 함께 걸려 들어옵니다. 이런 잡음을 그대로 두면 3장의 인과 추출이 엉뚱한 관계를 캐냅니다. 그래서 먼저 체로 한 번 거릅니다.

2. 판정 기준을 먼저 ‘글’로 정한다

코드를 짜기 전에, 무엇이 관련이고 무엇이 아닌지를 사람 말로 또렷이 정의합니다. 이 정의가 그대로 프롬프트가 됩니다.

관련 있음 (relevant=1)관련 없음 (relevant=0)
물리적 안정성: 응집·점도·입자·혼탁mAb를 검출 도구로만 사용(ELISA 시약, 유세포)
화학적 안정성: 산화·탈아미드화·이성질화·단편화·전하변이안정성 내용 없는 임상 효능/시험
생물학적 안정성: 형태 안정성·Tm·결합활성·역가mAb와 무관한 다른 단백질의 안정성
제형 조건(부형제·pH·완충액·농도)의 영향안정성 초점 없는 제조 공정
스트레스(동결-융해·열·교반·빛)의 영향, 시험법(SEC·DLS·DSC…)
불안정성의 하류 결과: 면역원성·ADA·PK 변화·안전성

3. 기준을 시스템 프롬프트로 옮기기

위 기준을 GPT의 시스템 프롬프트에 그대로 넣고, 출력은 반드시 [{pmid, relevant, reason}] 형태의 JSON 배열로만 내도록 못 박습니다(파싱하기 쉽게).

cell · system prompt
SYSTEM_PROMPT = """You are an expert in monoclonal antibody (mAb) formulation and stability.

Your task: Judge whether each article contains MECHANISTIC information
relevant to mAb stability.

RELEVANT (relevant=true) if the article covers ANY of:
- Physical stability: aggregation, viscosity, particle formation, turbidity
- Chemical stability: oxidation, deamidation, isomerization, fragmentation, charge variants
- Biological stability: conformational stability, Tm, binding activity, potency, ADCC
- Effect of formulation (excipients, pH, buffer, concentration) on mAb stability
- Effect of stress (freeze-thaw, thermal, agitation, light) on mAb stability
- Stability testing methods (SEC-HPLC, DLS, DSC, CE-SDS, ...)
- Downstream consequences: immunogenicity, ADA, PK changes, safety

NOT RELEVANT (relevant=false) if:
- mAb used only as a detection tool (ELISA reagent, flow cytometry)
- Clinical efficacy/trial with no stability content
- Stability of non-mAb molecules

Respond ONLY with a JSON array, one object per article:
[{"pmid": "...", "relevant": true/false, "reason": "one sentence"}]
No markdown, no explanation outside the JSON array."""
프롬프트가 곧 ‘채점 기준표’ 결과가 마음에 안 들면 코드가 아니라 이 프롬프트를 고칩니다. “이런 것도 관련으로 봐줘/빼줘”를 한두 줄 추가하면 판정이 달라집니다.

4. 배치 판정 함수 — 10편을 한 번에

한 편씩 호출하면 느리고 비쌉니다. 10편을 한 묶음으로 보내 한 번에 판정하면 호출 수가 1/10로 줍니다. 실패하면 잠깐 쉬고 재시도하고, 끝내 실패하면 보수적으로 ‘관련(1)’ 처리해 흘려보내지 않습니다.

cell · filter_batch
from openai import OpenAI
import json, time
client = OpenAI(api_key=OPENAI_KEY)   # 키는 Colab Secret에서 (0장 참고)

BATCH_SIZE, MAX_RETRIES = 10, 3

def filter_batch(articles):
    """논문 묶음을 GPT에 보내 [{pmid, relevant, reason}, ...] 반환"""
    lines = [f"PMID: {a['pmid']}\nTitle: {str(a['title'])[:200]}\n"
             f"Abstract: {str(a['abstract'])[:500]}" for a in articles]
    user_prompt = "\n\n---\n\n".join(lines)

    for attempt in range(MAX_RETRIES):
        try:
            resp = client.chat.completions.create(
                model="gpt-4o-mini",           # 저비용 모델
                messages=[{"role": "system", "content": SYSTEM_PROMPT},
                          {"role": "user",   "content": user_prompt}],
                max_completion_tokens=2000)
            content = resp.choices[0].message.content.strip()
            content = content.replace("```json", "").replace("```", "").strip()
            return json.loads(content)
        except Exception as e:
            if attempt < MAX_RETRIES - 1:
                time.sleep(2 + attempt * 2)
            else:   # 끝내 실패 → 보수적으로 모두 관련 처리
                return [{"pmid": a["pmid"], "relevant": True,
                         "reason": "filter_failed"} for a in articles]

5. 실행 — step1 CSV를 읽어 판정하고 step2 CSV로 저장

1장이 만든 CSV를 불러와, 묶음마다 판정하고, 결과 컬럼(relevant·reason)을 붙여 저장합니다.

cell · run filtering (CSV)
import pandas as pd
from tqdm import tqdm

IN  = f"{PROJECT_DIR}/step1_pubmed_abstracts.csv"
OUT = f"{PROJECT_DIR}/step2_filtered.csv"

df = pd.read_csv(IN)
records = df.to_dict("records")
results = {}

for i in tqdm(range(0, len(records), BATCH_SIZE), desc="Filtering"):
    for r in filter_batch(records[i:i+BATCH_SIZE]):
        results[str(r["pmid"])] = (int(bool(r.get("relevant"))), str(r.get("reason", ""))[:300])
    time.sleep(0.5)   # rate limit 예절

df["relevant"] = df["pmid"].astype(str).map(lambda p: results.get(p, (1, ""))[0])
df["reason"]   = df["pmid"].astype(str).map(lambda p: results.get(p, (1, ""))[1])
df.to_csv(OUT, index=False, encoding="utf-8-sig")

n_rel = int(df["relevant"].sum())
print(f"✅ 관련 {n_rel}편 / 제외 {len(df)-n_rel}편 → {OUT}")

6. 결과 점검 — 눈으로 확인하기

비율과 몇 개 샘플의 판정 이유를 직접 보며 프롬프트가 잘 작동했는지 확인합니다.

cell · sanity check
print(df["relevant"].value_counts())
print("\n--- 관련 샘플 ---")
for _, r in df[df.relevant==1].head(3).iterrows():
    print(f"[{r.pmid}] {str(r.title)[:70]}\n  → {r.reason}")
print("\n--- 제외 샘플 ---")
for _, r in df[df.relevant==0].head(3).iterrows():
    print(f"[{r.pmid}] {str(r.title)[:70]}\n  → {r.reason}")

한 단계 더 — 규모를 키우려면 DB

원본 프로젝트는 CSV 대신 MySQL(mab_pubmed_filtered 테이블)에 저장합니다. 수만 편을 여러 사람이 ‘이어하기’ 할 때 유리하죠. 핵심은 똑같습니다 — 입력 테이블을 읽고, 판정하고, 이미 처리한 PMID는 건너뜁니다. 무료로 연습하려면 DB 대신 SQLite(파일 한 개)를 써도 됩니다(부록 F).

접속정보는 코드에 적지 않기 DB를 쓸 때 주소·아이디·비밀번호는 Colab Secret/환경변수로 불러옵니다. 공개 저장소에 올라간 평문 비밀번호는 사고의 단골 원인입니다.
비용 감각 gpt-4o-mini + 10편 배치면 ~2,000편을 매우 저렴하게 처리합니다. 더 정확히 하고 싶을 때만 상위 모델로 바꾸면 됩니다.

  • 10편 배치 + 재시도로 싸고 안정적으로 판정했습니다.
  • step2_filtered.csv(relevant·reason 포함)를 얻었습니다.
