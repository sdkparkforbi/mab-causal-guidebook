그림 그리기 + 시각화 앱 (Vercel)

3장의 엣지 CSV를 인터랙티브 네트워크로 띄우고, 내 GitHub → Vercel로 세상에 공개합니다.

이 장의 결과물 https://<프로젝트>.vercel.app — 노드를 끌고 필터를 움직이며 탐색하는, 내 계정으로 배포한 살아있는 인과 네트워크 사이트.
라이브 데이터 = GPT-5-mini (정본) 실제 배포된 mab-causal-network.vercel.app는 GPT-5-mini 파이프라인의 32,939 엣지 · 8,670 노드를 보여줍니다(이전 4o-mini 16,019 엣지에서 교체). 데이터는 GitHub raw의 step2c_v4c_causal_edges_summary.csv를 직접 읽으므로, 그 파일만 바꿔 push하면 갱신됩니다.

0. 작업 흐름 — 클론 → 내 GitHub 업로드 → 배포

원본 시각화 저장소를 내 PC로 클론한 뒤, 내 GitHub의 새 repo로 올리고, 그 repo를 Vercel에 연결합니다. (남의 repo는 클론만으로 배포할 수 없습니다 — 내 소유여야 합니다.)

1. 읽는 법 — 색과 크기의 약속

  • 노드 크기 = 등장 빈도, 화살표 두께 = 관계 빈도(num_papers)
  • 화살표 색: 녹색 안정화 빨강 불안정화 회색 중립
  • 노드 색 = 6개 카테고리(sequence/structure/formulation/stress/stability/quality_outcome)

2. 내 데이터로 교체하기

원본 repo에는 예시 엣지 CSV가 들어 있습니다. 이걸 3장에서 내가 만든 step3_edges.csv로 바꾸면 “내 그래프”가 됩니다. 파일을 repo 루트에 넣고, index.html이 읽는 파일명·컬럼명만 맞춰주면 끝입니다.

컬럼 약속 cause, effect, relationship, frequency, num_papers, sample_evidence — 3장 출력과 동일합니다. (카테고리 색을 쓰려면 category_cause, category_effect도 함께)

3. 핵심 코드 — 엣지 CSV로 네트워크 그리기

가벼운 vis-network와 PapaParse(CSV 파서)를 CDN으로 불러와, 브라우저에서 바로 그립니다. 서버가 필요 없습니다.

index.html (발췌)
<script src="https://unpkg.com/vis-network/standalone/umd/vis-network.min.js"></script>
<script src="https://unpkg.com/papaparse@5/papaparse.min.js"></script>
<div id="net" style="height:80vh"></div>
<script>
// 관계유형 → 극성 색
const POS = ["stabilizes","inhibits","prevents","decreases","shields"];
const NEG = ["destabilizes","promotes","increases","induces","aggregates",
             "oxidizes","deamidates","fragments","unfolds","degrades"];
const color = r => POS.includes(r) ? "#2e9e6b" : NEG.includes(r) ? "#d65a4e" : "#9aa7b0";

// 필터 기준 (UI 슬라이더와 연결)
let MIN_FREQ = 3, MIN_PAPERS = 3;

Papa.parse("step3_edges.csv", { download:true, header:true, complete: res => {
  const nodes = {}, edges = [];
  res.data.forEach(e => {
    if (!e.cause || !e.effect) return;
    if (+e.frequency < MIN_FREQ || +e.num_papers < MIN_PAPERS) return;
    [e.cause, e.effect].forEach(n => {
      if (!nodes[n]) nodes[n] = { id:n, label:n, value:1 };
      else nodes[n].value++;
    });
    edges.push({ from:e.cause, to:e.effect, value:+e.num_papers,
                 color:color(e.relationship), title:e.relationship+": "+e.sample_evidence,
                 arrows:"to" });
  });
  new vis.Network(net, { nodes:Object.values(nodes), edges },
    { physics:{ stabilization:true }, nodes:{ shape:"dot", scaling:{ min:6, max:40 } } });
}});
</script>
호버하면 근거가 뜬다 엣지의 title에 3장에서 저장한 sample_evidence를 넣었습니다. 화살표에 마우스를 올리면 “이 관계가 어느 문장에서 나왔는지”가 바로 보입니다.

4. 필터 UI — 복잡도 조절

3.3만 엣지를 다 그리면 ‘털뭉치’가 됩니다. 최소 빈도·최소 논문수 슬라이더로 굵은 줄거리만 남깁니다.

index.html · 필터
최소 빈도 <input type="range" min="1" max="20" value="3"
  oninput="MIN_FREQ=+this.value; redraw()">
최소 논문수 <input type="range" min="1" max="20" value="3"
  oninput="MIN_PAPERS=+this.value; redraw()">
// redraw(): 위 Papa.parse 블록을 함수로 묶어 다시 호출
대응 자료 · 기술 선택 이 그림책은 배우기 쉬운 vis-network로 그리지만, 실제 연구 사이트는 D3.js force-directed로 구현돼 있습니다(원리는 동일 — 노드 간 인력·척력 시뮬레이션). 완성형 참고: sdkparkforbi.github.io/mab-causal-network, mab-causal-network-v2. 막히면 원본 index.html과 비교해 보세요.

5. Vercel로 배포

1

로그인

vercel.com에 내 GitHub로 로그인.

2

Import

Add New ▸ Project ▸ Import에서 내 mab-causal-network2 선택.

3

설정

Framework = Other, Root = index.html이 있는 폴더.

4

Deploy

.vercel.app URL 발급. 이후 git push만 하면 자동 재배포.

데이터를 바꾸면? step3_edges.csv를 새로 덮어쓰고 git push → 몇 초 뒤 사이트가 새 그래프로 갱신됩니다. (GitHub Pages로 올리려면 부록 C 참고)

오늘의 마무리

  • 원본을 클론 → 내 GitHub repo로 업로드하는 흐름을 익혔습니다.
  • 엣지 CSV를 vis-network로 그리고, 필터로 복잡도를 조절했습니다.
  • Vercel로 내 계정에서 사이트를 공개했습니다.

그래프를 ‘보는’ 앱은 만들었습니다. 5장에서는 그 내용이 ‘맞는지’ 검증하는 앱을 Streamlit으로 만듭니다.
