CHAPTER 5

검증 앱 (HITL · Streamlit + DB)

AI가 만든 기전 해석을 전문가가 검토하고, 결과를 DB에 저장합니다 — 정본 논문의 세 번째 부품(HITL)입니다.

이 장의 결과물 https://<앱이름>.streamlit.app — 빈도 가중 300개 샘플의 GPT 해석을 리뷰어가 적절/부적절 + 의견으로 판정하고, 판정 결과가 중앙 DB에 쌓입니다.
왜 Streamlit + DB인가 여러 리뷰어의 판정을 한 곳(DB)에 모아야 적절율을 신뢰성 근거로 쓸 수 있습니다. 정적(localStorage)은 한 사람 브라우저에만 남아 이 목적에 맞지 않습니다. 그래서 이 앱은 파이썬 서버형(Streamlit)으로, Streamlit Cloud에 배포합니다. (정적 앱은 Vercel, 이 앱만 Streamlit Cloud)

1. 왜 검증인가 — 휴먼 인 더 루프

GPT가 뽑은 해석엔 오류가 섞일 수 있습니다. 모델을 믿기 전에 표본을 전문가가 확인해 품질을 수치로 증명합니다. 이 “적절율(%)”이 정본 논문의 정성 검증 근거가 됩니다.

2. 앱 구조 — Streamlit + 두 DB 테이블

3. 핵심 코드 — DB에 판정 저장

비밀값은 st.secrets로 읽고, 판정은 mab_review_results에 INSERT/UPDATE 합니다.

app.py (발췌)
import streamlit as st
from sqlalchemy import create_engine, text

DB_URL = st.secrets["DB_URL"]              # 코드에 키 없음 (Cloud Secrets)
engine = create_engine(DB_URL)

# 샘플 로드
samples = pd.read_sql("SELECT sample_id, target_node, target_subcat, "
                      "record_json, gpt_interpretation FROM mab_review_samples", engine)

# 한 건 보여주고 판정 받기
st.info(row.gpt_interpretation)
verdict  = st.radio("이 해석이 타당한가요?", ["적절", "부적절"], horizontal=True)
feedback = st.text_area("의견 (선택)")

if st.button("저장") and reviewer:
    with engine.begin() as c:
        c.execute(text("INSERT INTO mab_review_results "
            "(sample_id, reviewer, verdict, feedback) "
            "VALUES (:s, :r, :v, :f)"),
            {"s":row.sample_id, "r":reviewer, "v":verdict, "f":feedback})
    st.success("DB에 저장되었습니다 ✓")

6. 적절율 집계

집계 (SQL)
SELECT verdict, COUNT(*) FROM mab_review_results GROUP BY verdict;
-- 적절율 = 적절 / 전체 → 예) 적절율 86%
이 숫자가 논문이 된다 “300 expert-curated explanations이 검토 대기/완료”가 정본 논문의 정성 검증 절을 채웁니다. 부적절 의견은 3장 프롬프트 개선으로 되먹입니다.

다른 길 — DB가 필요 없을 때(정적 lite)

오프라인 lite 버전 DB 없이 한 사람이 빠르게 검토만 하려면, 정적 버전 mab-causal-review(localStorage·Vercel)도 있습니다. 단, 결과가 브라우저에만 남으므로 여러 리뷰어 집계·DB 저장이 필요하면 이 장의 Streamlit 버전을 쓰세요.

오늘의 마무리

  • 전문가 검토(HITL)를 Streamlit + DB로 구현해 판정을 중앙 저장했습니다.
  • 키는 st.secrets로 안전하게, 결과는 mab_review_results에 INSERT.
  • 적절율로 모델 신뢰성을 수치화했습니다.
