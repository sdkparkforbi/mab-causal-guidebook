CHAPTER 5

검증 앱 (Vercel + DB + 대시보드)

AI가 만든 기전 해석을 전문가가 검토하고, 결과를 DB에 저장하고, 대시보드로 한눈에 봅니다.

이 장의 결과물 · 실제 배포됨 검증 → mab-causal-review.vercel.app · 대시보드 → /dashboard.html. 판정이 MySQL에 저장되고, 대시보드가 적절율·표를 보여줍니다.
구조 한 줄 — Vercel로 통일하면서 DB까지 정적 화면만으로는 DB 저장이 안 됩니다. 그래서 정적 프론트 + Vercel 서버리스 함수(/api)를 함께 둬, 프론트가 /api/review로 판정을 보내면 함수가 MySQL에 INSERT합니다. 모든 앱을 Vercel 하나로 통일하면서 중앙 DB도 갖춥니다.

1. 왜 검증인가 — 휴먼 인 더 루프

GPT 해석엔 오류가 섞일 수 있습니다. 전문가가 표본을 확인해 품질을 수치로 증명하고, 결과를 한 DB에 모아 적절율을 신뢰성 근거로 씁니다.

2. 앱 구조 — 정적 + 서버리스 함수

mab-causal-review/

3. 핵심 코드 — DB에 저장하는 서버리스 함수

api/review.js (발췌)
import mysql from 'mysql2/promise';

export default async function handler(req, res) {
  if (req.method !== 'POST') return res.status(405).json({error:'POST only'});
  const b = req.body || {};
  // 접속정보는 코드에 없음 → Vercel 환경변수
  const conn = await mysql.createConnection({
    host:process.env.DB_HOST, port:+process.env.DB_PORT,
    user:process.env.DB_USER, password:process.env.DB_PASSWORD,
    database:process.env.DB_NAME, charset:'utf8mb4' });
  await conn.execute(`INSERT INTO mab_causal_review_results
      (sample_id,reviewer,verdict,feedback,cause,effect,relationship)
      VALUES (?,?,?,?,?,?,?)`,
    [b.sample_id,b.reviewer,b.verdict,b.feedback,b.cause,b.effect,b.relationship]);
  await conn.end();
  res.status(200).json({ok:true});
}

4. DB 환경변수 — 코드에 키를 두지 않는다

접속정보는 Vercel ▸ Settings ▸ Environment Variables에 넣습니다(암호화 저장). 코드는 process.env로만 읽습니다.

변수예시
DB_HOST106.247.236.2
DB_PORT3306
DB_USER / DB_PASSWORDuser / ••••
DB_NAMEabswitch
DB는 외부 접속 허용 필요 Vercel 함수가 연결하려면 MySQL이 외부 IP 접속을 허용해야 합니다. 막혀 있으면 무료 서버리스 DB(Vercel Postgres·Supabase)로 바꾸면 됩니다.
