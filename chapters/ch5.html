검증 앱 만들기 (Streamlit)

AI가 뽑은 인과관계가 정말 타당한지, 사람이 검수하는 웹앱을 만들고 Streamlit Cloud로 공개합니다.

이 장의 결과물 https://<앱이름>.streamlit.app — 리뷰어가 샘플별로 ‘적절/부적절’ + 의견을 남기고, 그 결과로 모델 신뢰성을 수치화하는 앱.

1. 왜 검증인가 — 휴먼 인 더 루프

GPT가 뽑은 인과관계에는 오류가 섞일 수 있습니다. 모델을 믿기 전에 표본을 사람이 확인해 품질을 수치로 증명해야 합니다. 이 “적절율(%)”은 8장 논문에서 신뢰성의 근거가 됩니다.

3. 핵심 코드 — 리뷰 화면

샘플을 불러와 한 건씩 보여주고, 리뷰어가 적절/부적절 + 의견을 남기면 저장합니다. 비밀값은 코드에 적지 않고 st.secrets로 읽습니다.

app.py (발췌)
import streamlit as st
import pandas as pd

st.title("🧪 mAb 인과관계 검증")

# 비밀값은 Secrets에서 (코드/repo에 절대 평문 금지)
# DB_URL = st.secrets["DB_URL"]   # 예: mysql+pymysql://...

# 검증 샘플 로드 (DB 또는 CSV)
@st.cache_data
def load_samples():
    return pd.read_csv("review_samples.csv")   # 무료 경로: CSV 300건

df = load_samples()
name = st.text_input("리뷰어 이름")
sub  = st.selectbox("하위범주", ["전체", "physical", "chemical", "biological"])
view = df if sub == "전체" else df[df.subcat == sub]

i = st.number_input("샘플 번호", 0, len(view)-1, 0)
row = view.iloc[int(i)]

st.markdown(f"**그래프:** {row.cause} → {row.effect}  ({row.relationship})")
st.info(f"GPT 해석: {row.interpretation}")

verdict  = st.radio("이 해석이 타당한가요?", ["적절", "부적절"], horizontal=True)
feedback = st.text_area("의견 (선택)")

if st.button("저장") and name:
    rec = pd.DataFrame([{"reviewer":name, "sample_id":row.id,
                         "verdict":verdict, "feedback":feedback}])
    rec.to_csv("reviews.csv", mode="a", header=False, index=False)
    st.success("저장되었습니다 ✓")
DB로 확장 여러 리뷰어가 동시에 쓰려면 CSV 대신 DB(MySQL)에 저장합니다. 접속 문자열은 st.secrets["DB_URL"]로 읽고, 실제 값은 Streamlit Secrets 화면에만 둡니다.

5. 결과 집계 — 적절율 구하기

집계
r = pd.read_csv("reviews.csv", names=["reviewer","sample_id","verdict","feedback"])
rate = (r.verdict == "적절").mean()
print(f"적절율: {rate:.1%}  (n={len(r)})")   # 예) 적절율: 86.0% (n=300)
이 숫자가 논문이 된다 “리뷰어 검증 결과 적절율 86%” 같은 한 줄이 8장 Results의 신뢰성 절을 채웁니다. 부적절로 표시된 사례의 의견은 프롬프트 개선(3장)으로 되먹임할 수 있습니다.
