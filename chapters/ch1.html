PubMed에서 초록 수집하기

우리 인과 지도의 원재료인 논문 초록 수천 편을, 키워드로 검색해 내려받습니다.

이 장의 결과물 step1_pubmed_abstracts.csv — 약 2,000여 편의 pmid · 제목 · 초록 · 연도 · 저널이 담긴 표 한 장. 이것이 2장(필터링)의 입력이 됩니다.
실제 연구 규모 여기서는 키워드 15개로 ~2,100편을 모으는 “미니 버전”을 돌립니다. 실제 논문 파이프라인은 아래 7개 인과 사슬 카테고리에 걸친 126개 쿼리로 62,281편(1980–2026)을 누적 수집했습니다. 검색 원리는 똑같으니, 규모만 키우면 됩니다.

1. 큰 그림: 검색하고(eSearch) 받아오기(eFetch)

PubMed은 미국 국립의학도서관(NCBI)이 운영하는 의학 논문 데이터베이스입니다. 코드로 접근할 때는 E-utilities라는 공개 API를 씁니다. 두 단계만 기억하면 됩니다.

  • eSearch: 키워드를 던지면 → 조건에 맞는 논문들의 고유번호(PMID) 목록을 줍니다.
  • eFetch: PMID 목록을 던지면 → 각 논문의 제목·초록·연도를 XML로 줍니다.

4. 검색 키워드 설계 — 왜 15개나?

“monoclonal antibody AND stability” 한 줄만으로는 놓치는 논문이 많습니다. mAb를 부르는 다른 말(therapeutic antibody, IgG, bispecific…), 안정성을 가리키는 다른 말(aggregation, degradation, shelf life…)을 골고루 섞어 그물을 넓게 칩니다.

검색어 설계가 곧 데이터 품질 나중에 키워드를 추가하고 이 노트북만 다시 돌리면 데이터가 늘어납니다. “핵심 조합 → 동의어 확장”의 2단 구성을 기억하세요.

실제 논문은 검색어를 인과 사슬(sequence → structure → formulation·stress → stability → quality → PK/PD)을 따라 7개 카테고리·126개 쿼리로 확장했습니다. 미니 버전을 키울 때 참고하세요.

2. Structure"conformational stability", "Fab domain", "Fc region", "tertiary structure"
3. Formulation"excipient", "pH", "polysorbate", "histidine buffer", "sucrose"
4. Stress"thermal stress", "agitation", "freeze-thaw", "oxidative stress"
5. Stability"aggregation", "viscosity", "oxidation", "deamidation", "fragmentation"
6. Quality Outcome"immunogenicity", "potency", "subvisible particles", "turbidity"
7. PK/PD/Safety"pharmacokinetics", "bioavailability", "half-life", "clearance"

5. 크롤링 함수 두 개

앞서 본 eSearch · eFetch를 각각 함수로 감쌉니다.

(1) search_pubmed — 키워드 → PMID 목록

cell 4a · search_pubmed
def search_pubmed(query, max_results=200):
    """PubMed eSearch → PMID 목록 반환"""
    url = "https://eutils.ncbi.nlm.nih.gov/entrez/eutils/esearch.fcgi"
    params = {
        "db": "pubmed", "term": query,
        "retmax": max_results, "retmode": "json",
        "email": PUBMED_EMAIL,
    }
    resp = requests.get(url, params=params); resp.raise_for_status()
    data = resp.json()
    pmids = data.get("esearchresult", {}).get("idlist", [])
    total = int(data.get("esearchresult", {}).get("count", "0"))
    return pmids, total

(2) fetch_abstracts — PMID → 제목·초록·연도

PMID를 200개씩 묶어 한 번에 받아옵니다(배치). 받은 XML에서 제목·초록·연도·저널을 꺼내고, 초록이 너무 짧은(50자 미만) 항목은 버립니다. NCBI 예절을 지키려 요청 사이에 잠깐 쉽니다(time.sleep).

cell 4b · fetch_abstracts
def fetch_abstracts(pmids):
    """PubMed eFetch → title, abstract, year 추출"""
    url = "https://eutils.ncbi.nlm.nih.gov/entrez/eutils/efetch.fcgi"
    articles, batch_size = [], 200

    for i in tqdm(range(0, len(pmids), batch_size), desc="배치 다운로드"):
        batch = pmids[i:i+batch_size]
        params = {"db": "pubmed", "id": ",".join(batch),
                  "rettype": "xml", "retmode": "xml", "email": PUBMED_EMAIL}
        resp = requests.get(url, params=params); resp.raise_for_status()
        parsed = xmltodict.parse(resp.text)

        items = parsed.get("PubmedArticleSet", {}).get("PubmedArticle", [])
        if isinstance(items, dict): items = [items]

        for art in items:
            try:
                medline = art["MedlineCitation"]
                pmid = medline["PMID"]["#text"] if isinstance(medline["PMID"], dict) else medline["PMID"]
                a = medline["Article"]
                title = a.get("ArticleTitle", "")
                if isinstance(title, dict): title = title.get("#text", "")

                # 초록은 여러 단락(Label)일 수 있어 합쳐 줍니다
                ab = a.get("Abstract", {}).get("AbstractText", "")
                if isinstance(ab, list):
                    parts = []
                    for p in ab:
                        if isinstance(p, dict):
                            lab, txt = p.get("@Label", ""), p.get("#text", "")
                            parts.append(f"{lab}: {txt}" if lab else txt)
                        else: parts.append(str(p))
                    abstract = " ".join(parts)
                elif isinstance(ab, dict): abstract = ab.get("#text", "")
                else: abstract = str(ab)

                year = a.get("Journal",{}).get("JournalIssue",{}).get("PubDate",{}).get("Year", "N/A")
                journal = a.get("Journal", {}).get("Title", "N/A")

                if abstract and abstract != "None" and len(abstract) > 50:
                    articles.append({"pmid": pmid, "title": title,
                                     "abstract": abstract, "year": year, "journal": journal})
            except (KeyError, TypeError):
                continue
        time.sleep(0.4)   # NCBI rate limit 준수
    return articles

6. 크롤링 실행

15개 쿼리를 차례로 검색해 PMID를 모으고(중복 자동 제거), 그 PMID들의 초록을 한꺼번에 받습니다.

cell 5 · run search
all_pmids = set()
for i, query in enumerate(SEARCH_QUERIES, 1):
    pmids, total = search_pubmed(query, max_results=MAX_RESULTS_PER_QUERY)
    new = set(pmids) - all_pmids
    all_pmids.update(pmids)
    print(f"[{i:2d}] 검색 {len(pmids)}건 (전체 {total}건), 신규 {len(new)}건")
    time.sleep(0.5)
print(f"📊 중복 제거 후 총 PMID: {len(all_pmids)}건")

막힐 때 — 자주 보는 문제

증상원인 / 해결
HTTP 429 또는 응답 지연요청이 너무 빠름 → time.sleep 값을 키우세요. (이메일 기입 권장)
초록이 비어 수집 수가 적음일부 논문은 초록이 없음 → 정상. 50자 미만 필터 때문에 제외됩니다.
CSV 한글·기호 깨짐저장 시 encoding="utf-8-sig" 확인.
Drive가 다시 마운트 요구런타임이 끊기면 재마운트 필요 → cell 2 다시 실행.
